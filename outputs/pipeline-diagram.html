ai-os-v2

A project launcher for AI coding agents. Takes a raw idea through structured alignment and produces a self-governing project repo.

Pipeline

1
align
Grill the user on their idea. Resolve domain terms, derive tech stack from requirements, lock decisions.
context.md, align.md
›
2
to-prd
Synthesize a PRD from alignment outputs. No re-interviewing — just synthesis.
prd.md
›
3
to-issues
Decompose PRD into vertical slice issues organized by phase and dependency.
issues/ directory
›
4
validate-slices
Audit each issue against 7 checks: user value, layer coverage, demoability, dependency shape, size, phase 0 leakage, acceptance-test clarity.
slice-audit.md
›
5
handoff
Create project repo, copy artifacts, install project-mode skills, seed vendored runtime, push to GitHub.
~/ai-projects/<name>/

After handoff: cd ~/ai-projects/<name>/ and run /to-sandcastle from the project repo.

Sandcastle Agent Loop

Each iteration picks unblocked issues from the current phase, works them in parallel Docker containers, then creates PRs.

Plan
Opus 4.7
Build dependency graph, pick unblocked issues, assign branches
›
Implement
GPT-5.5 low / Composer 2.5
One Docker container per issue, TDD with RGR loop
›
Review
GPT-5.5 high
Code quality pass, standards enforcement, build check
›
Create PRs
Opus 4.7
Push branches, create PRs with Closes #N, remove labels
›
Merge
Strategy fork
Auto-merge or wait for human

AUTO Auto-merge

Script runs typecheck, test, build on each PR. If all pass: approve + squash merge. If any fail: fall back to human review. Loop continues to next phase.

HUMAN Human-merge

Script pauses after PRs are created. Polls every 30s until all PRs show state MERGED. Then continues to next phase.
